# 🧾 당근마켓 클론코딩 단위 프로젝트 보고서

## 📝 1. 기획서

### 📌 클론할 사이트 선정 이유
사용자 간 중고 거래를 중심으로 한 실시간 커뮤니티 플랫폼이라는 점에서 활용도가 높고, 주요 기능이 다양하게 구성되어 있어 리액트 학습에 적합하다고 판단 했습니다.

### 🌐 참고한 원본 사이트
- **사이트명**: 당근마켓  
- **주소**: [https://www.daangn.com](https://www.daangn.com)  
- **스크린샷**:  
  ![스크린샷 예시](./images/daangn_main.png)

### 📄 클론할 페이지

- 헤더/푸터, 검색창
- 메인 페이지
- 중고거래 목록 페이지
- 중고거래 상세 페이지
- 동네생활 목록 페이지
- 동네생활 상세 페이지
- 앱 다운로드 권유 QR 모달창
- 에러페이지

### 🎯 프로젝트 목표
- React를 활용한 SPA 구조 학습  
- 실제 서비스 수준의 UI/UX 구현  
- Context API 및 React Hook 사용
- ##화기애애한 분위기, 많은 실패 경험.

### 🗓 일정 계획

| 주차 | 진행 내용 |
|------|-----------|
| 1일차 | 사이트 분석 역할 분담. 기능 구현 시 |
| 2일차 | 레이아웃 구성 및 주요 컴포넌트 구현 |
| 3일차 | 기능 개발 및 API 연결, 테스트, 배포 및 마무리 문서 작성 |

## 📋 2. 요구사항 정의서

### 사용자 요구사항

| 항목 | 설명 |
|------|------|
| 회원가입/로그인 | 사용자는 이메일/비밀번호로 로그인 및 가입 가능해야 함 |
| 상품 등록 | 사용자는 이미지를 첨부하여 상품 등록 가능 |
| 상품 조회 | 메인 페이지에서 상품 리스트를 확인 가능 |
| 상품 상세보기 | 상품 클릭 시 상세 페이지로 이동 |
| 채팅 기능 (선택사항) | 구매자-판매자 간 실시간 메시지 기능 (개발 여부에 따라 유동적) |
| 항목 | 설명 |
| ** 상품 탐색:** |  |
| 상품 탐색 기능 | 홈페이지에서 최신 상품 또는 인기 상품 목록 확인 가능 |
| 특정 키워드 상품 검색 기능 | 사용자는 원하는 상품을 빠르게 찾기 위해 검색창에 키워드를 입력하여 검색할 수 있어야 합니다. |
| 카테고리별 상품 목록 확인 기능 | 사용자는 관심 있는 상품의 종류별로 분류된 목록을 확인할 수 있어야 합니다. |
| 검색 범위 조정 기능 | 사용자는 다양한 필터를 이용하여 원하는 조건에 맞는 상품을 쉽게 찾을 수 있도록 상품 목록을 정렬하거나 검색 범위를 좁힐 수 있어야 합니다. |
| 상품 간략 정보 확인 기능 | 사용자는 상품 목록 페이지에서 각 상품의 기본적인 정보(이미지, 제목, 가격, 판매 지역)를 한눈에 파악할 수 있어야 합니다. |
| 더보기  | 사용자는 초기 로딩된 상품 외에 추가적인 상품 목록을 보기 위해 "더보기" 버튼 등의 기능을 이용할 수 있어야 합니다. |
| **상품 상세 정보 확인** |  |
| 상품 상세 정보 확인 가능 | 사용자는 관심 있는 상품을 클릭하여 해당 상품의 상세 페이지로 이동하여 더 자세한 정보를 확인할 수 있어야 합니다. |
| 판매자의 다른 판매 상품 목록 확인 가능 | 사용자는 현재 보고 있는 상품의 판매자가 판매하는 다른 상품들의 목록을 확인할 수 있어야 합니다. |
| 인기 매물 목록 확인 가능 | 사용자는 현재 인기 있는 다른 상품들의 목록을 확인할 수 있어야 합니다. |
| **상품 목록 표시** |  |
| 상품 목록 페이지에서 상품 이미지, 제목, 가격, 위치, 등록 시간 등 필요한 정보 간략 표시 | 시스템은 상품 목록 페이지에서 각 상품에 대한 필수적인 정보(이미지, 제목, 가격, 판매 지역, 등록된 시간)를 간결하게 표시해야 합니다. |
| "더보기" 기능을 통해 대량의 상품 목록 효율적 표시 | 시스템은 많은 수의 상품을 한 번에 로딩하는 대신, 사용자의 요청에 따라 추가적으로 상품 목록을 로딩하여 페이지 로딩 속도를 개선하고 사용자 경험을 향상시켜야 합니다. |
| **상품 상세 정보 표시** |  |
| 상품 이미지 제공 및 여러 장의 이미지를 슬라이드 형태로 보기 지원 | 시스템은 상품 상세 페이지에서 해당 상품의 이미지를 제공하고, 여러 장의 이미지가 등록된 경우 사용자가 슬라이드 형태로 편리하게 넘겨보며 확인할 수 있도록 지원해야 합니다. |
